## 非機能要件(品質)への考慮

* テスト(不具合) => **保守性**
* パフォーマンス(負荷分散・冗長化) => **性能**
	* ロードバランシング、スケーラビリティ
* セキュリティ => **安全性**
* モニタリング(分析・監視・障害・復旧) => **信頼性**

=> ソフトを高速、安定、安全に運用し、運営に役立つデータを可視化すること
=> 品質面は、考慮するが、機能定義はしない

## 構成要素

* 機能カタログ(+CSDI) => **(1)機能性**
* 画面デザイン(PWCS) => **(2)使用性**
* 品質チェック(TPSM) => **(3)品質性**

=> 全体を通して、発注者の「検収コスト」の削減が必須
